Description

Composer is a PHP package dependency manager that XOOPS depends on. You can find out more about Composer from getcomposer.org.

Normally Composer is executed from a command line, but there are situations where this is not possible or desired. For these situations Composer Tool provides a simple interface for basic Composer tasks in the form of a XOOPS Extension.

Usage

The Composer Tool is available in the XOOPS administration area. Select the Composer tab, and you will be presented with a list of common composer commands to select from. The most common are listed in the Basics section of the list. These commands are:

  • Check for Updates - check for updated packages
  • Update - update packages as specified in composer.json
  • Optimize autoloader - builds a highly optimized classmap for the autoloader. While this can speed up site performance, it can complicate development work. You should run this for a production system after each update.
  • Update Composer - get the latest version of Composer. This needs to be done periodically as Composer may issue errors if it is significantly out of date.

Security Check

The security check function will check any Composer installed packages against the SensioLabs security advisories database and issue a report of any relevant advisories. For more information on the SensioLabs Security Advisory Database, see https://security.sensiolabs.org/.
